
# 🚀 راه‌اندازی VLESS با GitHub Codespaces

یک روش ساده برای اجرای **VLESS Proxy** با استفاده از **GitHub Codespaces**.  
اگر بتوانید Codespaces را باز کنید، می‌توانید از این پروژه استفاده کنید.

---

## 📦 مراحل راه‌اندازی

1. این ریپازیتوری را **Fork** کنید.
2. روی دکمه سبز **Code** در بالای صفحه کلیک کنید.
3. وارد تب **Codespaces** شوید.
4. روی **Create codespace on main** کلیک کنید.

بعد از آن GitHub شروع به آماده‌سازی محیط می‌کند.

---

## ⚙️ نحوه استفاده

1. چند دقیقه صبر کنید تا Codespace کامل آماده شود.
2. وقتی آماده شد، به تب **Terminal** بروید.
3. لینک **VLESS** شما به صورت خودکار در ترمینال نمایش داده می‌شود.
4. لینک را کپی کرده و در یکی از کلاینت‌های زیر وارد کنید:

- **V2RayNG**
- **V2RayN**
- **Clash / Sing-box / یا هر کلاینتی که از VLESS پشتیبانی می‌کند**

بعد از آن می‌توانید از پراکسی استفاده کنید ✅

---

## ⏱️ محدودیت استفاده

GitHub Codespaces در هر ماه **۱۲۰ ساعت CPU رایگان** ارائه می‌دهد.

مثال:

- ۱ هسته CPU → حدود **۱۲۰ ساعت**
- ۲ هسته CPU → حدود **۶۰ ساعت**
- ۴ هسته CPU → حدود **۳۰ ساعت**

💡 **نکته:**  
وقتی از Codespace استفاده نمی‌کنید، آن را **Stop** کنید تا ساعت‌های رایگان شما هدر نرود.

---

## 🌐 شبکه‌های تست شده

این پروژه روی **Shecan (پلن رایگان)** تست شده است.

اگر بتوانید به یکی از IP های زیر دسترسی داشته باشید، احتمالاً برای شما هم کار می‌کند:

```
63.141.252.203
50.7.5.83
63.141.252.203
94.130.50.12
```

اگر این IP ها برای شما کار نکردند، دیتاسنتر یا ISP های دیگر را امتحان کنید.
